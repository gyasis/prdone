--- feature: 001-prd-visualizer created: 2026-05-07 status: in-progress --- # Feature Specification: PRD Visualizer Extension <!-- a VSCode extension that surfaces ~/dev/prd as a sidebar tree --> ## User Scenarios & Testing ### US1 — Find what I'm working on right now As an engineer juggling 3-5 active PRDs across worktrees, I want to open my editor and see which PRDs are stale, which are due for cleanup, and which I should resume — without remembering exact filenames. ### US2 — Resume a PRD without losing context When I click a PRD tile, I want a one-click checkout() that runs "/prd-checkout <id>" — pulling Jira, Confluence, Bitbucket comments, and the PRD body into a single re-orientation briefing. ## Acceptance Criteria - MUST render in 3 themes: Default Dark, Default Light, High Contrast - MUST read tier classification from ~/dev/prd/{scratch,archive,library}/ - MUST degrade gracefully when prd CLI is unreachable - Tiles older than 14d with no prd log activity render as stale
