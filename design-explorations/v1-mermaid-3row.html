PRD Network Graph — design variants

Four aesthetic treatments of the same 3-row mermaid hierarchy (parent → self+siblings → children) for the side-panel embedded graph in the PRD Visualizer VSCode extension. Pick one and reply with v1, v2, v3, or v4.

Variant 1 — Editorial mermaid

Thin 1px lines, generous whitespace, Newsreader serif titles, restrained neutrals with a single accent. Reads like a New Yorker explanatory diagram.

Variant 2 — Dense IDE

Geist Mono throughout, tight spacing, VSCode-charcoal background with neon role accents (cyan self, purple parent, green siblings, orange children). Tool-diagram density.

Variant 3 — Hand-drawn marker

Wavy SVG-displaced strokes, slight box rotation jitter, cream paper, Caveat handwriting. Reads like a whiteboard sketch — playful, low commitment.

Variant 4 — Kanban-card chunk

Each node is a full mini-card: tier glyph + slug + age, soft shadow, rounded corners. Continuous with the existing kanban — bigger nodes, more info per box.

Pick-the-variant checklist

  • Self box is clearly differentiated
  • Connectors are right-angle (no curves)
  • Hover state is legible in this aesthetic
  • Could I look at this 50× a day without fatigue?
